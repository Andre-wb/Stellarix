# `ios/VortexApp/` — iOS App Target

The tiny iOS app target that embeds the `Modules` Swift Package and launches it. Thin on purpose — everything real lives in `../Modules/Sources/`.

## Files

| File               | Role                                                                                 |
| ------------------ | ------------------------------------------------------------------------------------ |
| `VortexApp.swift`  | `@main App` — constructs `AppEnvironment` (from `Modules/Sources/App/`) and renders the root scene. |
| `Info.plist`       | Permission usage strings, URL schemes, `NSAppTransportSecurity`, background modes, export compliance flag for crypto. |
| `Assets.xcassets`  | App icon, launch image, brand colours.                                               |

## Build

Generated by xcodegen from `../project.yml`:

```bash
brew install xcodegen
cd ios && xcodegen && open Vortex.xcodeproj
```

## Permissions

Declared in `Info.plist`:

- **Camera** — voice / video calls.
- **Microphone** — voice notes, calls.
- **Photo Library** — avatar upload + file attachments.
- **Local Network** — LAN peer discovery (required in recent iOS).
- **Bluetooth** — BLE transport fallback.
- **Notifications** — requested at runtime after onboarding.

## App scene

```swift
@main
struct VortexApp: App {
    @StateObject private var env = AppEnvironment.live()
    var body: some Scene {
        WindowGroup {
            AppRootView()
                .environmentObject(env)
        }
    }
}
```

`AppRootView` lives in `Modules/Sources/App/` and picks between Bootstrap → Auth → Main screens based on environment state.

## Build settings

- Deployment target: iOS 17.
- Swift language version: 5.10.
- Bitcode: disabled (deprecated in Xcode 15).
- Enable hardened runtime for TestFlight.
